# Virtual Outfit Advisor (VOA)

A production-ready web application that helps users digitize their wardrobe, get rule-based outfit
recommendations, and plan travel packing lists using real weather data.

**No machine learning, computer vision, or complex AI is used anywhere in this project.** All
recommendation logic is deterministic, rule-based Python/Django code (see
`backend/recommendations/services.py` and `backend/travel/services.py`). The only third-party API
integration is a single Weather API (OpenWeatherMap) used exclusively by the Travel Planner.

## Features

- **Landing Page** — marketing overview of the product.
- **Authentication** — Register, Login (JWT), Forgot Password / Reset Password via emailed token.
- **Dashboard** — quick wardrobe stats and shortcuts.
- **Digital Wardrobe** — upload, edit, delete, search, and filter (category, season, occasion, color)
  wardrobe items with photos.
- **Outfit Recommendations** — generates an outfit from the user's own wardrobe using simple
  color/season/occasion matching rules; results are saved to history and can be favorited.
- **Travel Planner** — enter a destination + dates, fetches a live weather forecast, and returns a
  packing checklist and wardrobe-based outfit suggestions using deterministic rules.
- **Favorites** — saved outfits and recommendations in one place.
- **Recommendation History** — a filterable log of every recommendation ever generated.
- **User Profile** — edit personal details, avatar, style preference, and change password.

## Backend Setup

### 1. Prerequisites
- Python 3.11+
- PostgreSQL 14+
- A free [OpenWeatherMap](https://openweathermap.org/api) API key (for the Travel Planner)

### 2. Install & configure

```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# Edit .env: set SECRET_KEY, DB credentials, WEATHER_API_KEY, FRONTEND_URL
```

### 3. Create the PostgreSQL database

```sql
CREATE DATABASE voa_db;
CREATE USER voa_user WITH PASSWORD 'voa_password';
GRANT ALL PRIVILEGES ON DATABASE voa_db TO voa_user;
```

### 4. Run migrations & start the server

```bash
python manage.py makemigrations
python manage.py migrate
python manage.py createsuperuser
python manage.py seed_demo_data     # optional: creates demo@voa.local / DemoPass123!
python manage.py runserver
```

The API is now available at `http://localhost:8000/api/`. Admin panel: `http://localhost:8000/admin/`.

## Frontend Setup

### 1. Prerequisites
- Node.js 18+

### 2. Install & build

```bash
cd frontend
npm install
npm run build       # compiles Tailwind CSS -> public/assets/styles.css
                     # compiles TypeScript -> public/assets/js/**
npm run dev          # watches CSS/TS and serves the app on http://localhost:5173
```

Open `http://localhost:5173/index.html` (or the page you need under `/pages/`).

> `frontend/src/ts/config.ts` sets `API_BASE_URL` to `http://localhost:8000/api` by default —
> update it if your backend runs elsewhere.

## Running with Docker (recommended for production-like setup)

```bash
cp backend/.env.example backend/.env
# edit backend/.env with real values (SECRET_KEY, WEATHER_API_KEY, DEBUG=False, etc.)

docker compose up --build
```

This starts:
- `db` — PostgreSQL 16
- `backend` — Django/Gunicorn API on port `8000`
- `frontend` — Nginx serving the compiled static site on port `5173`

Run migrations & seed data inside the backend container if needed:

```bash
docker compose exec backend python manage.py migrate
docker compose exec backend python manage.py createsuperuser
docker compose exec backend python manage.py seed_demo_data
```

## API Overview

All endpoints are prefixed with `/api/`. Authenticated endpoints require an `Authorization: Bearer <access_token>` header.

| Endpoint                                    | Method     | Description                              |
|----------------------------------------------|-----------|-------------------------------------------|
| `/auth/register/`                             | POST      | Create a new account                      |
| `/auth/login/`                                | POST      | Log in, returns JWT access/refresh tokens |
| `/auth/token/refresh/`                        | POST      | Refresh an expired access token           |
| `/auth/logout/`                               | POST      | Blacklist the refresh token                |
| `/auth/profile/`                              | GET/PATCH | View or update the logged-in user profile |
| `/auth/change-password/`                      | POST      | Change password (requires old password)   |
| `/auth/forgot-password/`                      | POST      | Request a password reset email/link       |
| `/auth/reset-password/`                       | POST      | Reset password using emailed token        |
| `/wardrobe/items/`                            | GET/POST  | List/create wardrobe items (search & filter supported) |
| `/wardrobe/items/{id}/`                       | GET/PATCH/DELETE | Retrieve, update, delete an item   |
| `/wardrobe/items/{id}/mark_worn/`             | POST      | Increment wear count                      |
| `/wardrobe/items/{id}/toggle_favorite/`       | POST      | Toggle favorite flag                      |
| `/wardrobe/items/stats/`                      | GET       | Aggregate wardrobe stats for dashboard    |
| `/wardrobe/outfits/`                          | GET/POST  | List/create manually built outfits        |
| `/recommendations/generate/`                  | POST      | Generate a rule-based outfit recommendation |
| `/recommendations/history/`                   | GET       | List past recommendations (filterable)    |
| `/recommendations/history/{id}/save/`         | POST      | Mark a recommendation as saved            |
| `/travel/plans/`                              | GET/POST  | List trips / create a trip (fetches weather, builds packing list) |
| `/favorites/`                                 | GET/POST/DELETE | Manage favorited outfits/recommendations |

## Design Principles

- **No AI/ML/CV**: Recommendation and packing logic is implemented with plain conditional rules and
  simple color/category/season matching — fully transparent and explainable.
- **Single external API**: Only the Weather API is called from the backend; no other third-party
  services are used.
- **Security**: JWT authentication, password validation, per-user data isolation (all querysets are
  scoped to `request.user`), CORS restricted to the configured frontend origin.
- **Scalable structure**: Each Django app owns a single responsibility (accounts, wardrobe,
  recommendations, travel, favorites), making the codebase easy to extend.
